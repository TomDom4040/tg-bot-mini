# 🔄 Система резервного копирования бота

## 📋 Обзор

Эта система обеспечивает автоматическое создание бэкапов и быстрое восстановление бота в случае поломки.

## 🛠️ Компоненты системы

### 1. Скрипт бэкапа (`backup.sh`)
- Создает полный архив бота
- Включает код, данные, настройки
- Автоматически очищает старые бэкапы
- Безопасно останавливает и запускает бота

### 2. Скрипт восстановления (`restore.sh`)
- Быстрое восстановление из любого бэкапа
- Интерактивный выбор бэкапа
- Автоматическая установка зависимостей
- Проверка работоспособности после восстановления

### 3. Автоматические бэкапы (Cron)
- Каждые 6 часов автоматический бэкап
- Еженедельная очистка старых бэкапов
- Логирование процесса

## 🚀 Использование

### Создание бэкапа вручную
```bash
cd /root/Tg_BOTs/tg-bot-mini
./backup.sh
```

### Восстановление из бэкапа
```bash
cd /root/Tg_BOTs/tg-bot-mini

# Показать список бэкапов
./restore.sh list

# Восстановить из последнего бэкапа
./restore.sh latest

# Восстановить из конкретного бэкапа
./restore.sh 1
```

### Просмотр логов бэкапов
```bash
tail -f /root/Tg_BOTs/backups/backup.log
```

## 📁 Структура бэкапов

```
/root/Tg_BOTs/backups/
├── tg-bot-backup-YYYYMMDD_HHMMSS.tar.gz  # Архивы бэкапов
└── backup.log                            # Логи процесса
```

## ⚙️ Настройки

### Автоматические бэкапы
- **Частота**: Каждые 6 часов
- **Хранение**: 7 дней
- **Очистка**: Автоматическая

### Размер бэкапов
- **Типичный размер**: 3-5 MB
- **Содержимое**: Код + данные + зависимости
- **Сжатие**: gzip (высокая степень сжатия)

## 🔧 Команды для экстренного восстановления

### Быстрое восстановление (1 команда)
```bash
cd /root/Tg_BOTs/tg-bot-mini && ./restore.sh latest
```

### Проверка статуса бота
```bash
curl -I http://165.232.139.129:3000
```

### Просмотр логов
```bash
tail -f /root/Tg_BOTs/tg-bot-mini/server.log
```

## 🚨 Экстренные ситуации

### Бот не запускается
1. Проверьте логи: `tail -f /root/Tg_BOTs/tg-bot-mini/server.log`
2. Восстановите из бэкапа: `./restore.sh latest`
3. Проверьте зависимости: `npm install`

### Потеря данных
1. Остановите бота: `pkill -f "node index.js"`
2. Восстановите из бэкапа: `./restore.sh list` → выберите нужный
3. Запустите бота: `cd /root/Tg_BOTs/tg-bot-mini && nohup node index.js > server.log 2>&1 &`

### Проблемы с правами доступа
```bash
chmod +x /root/Tg_BOTs/tg-bot-mini/*.sh
chown -R root:root /root/Tg_BOTs/tg-bot-mini/
```

- Бэкапы содержат все данные, включая пароли
- Храните бэкапы в безопасном месте
- Регулярно проверяйте целостность бэкапов
- Не делитесь бэкапами с третьими лицами
