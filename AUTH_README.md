# Система входа по Email и паролю

## Что было добавлено

Теперь в админку можно входить по email и паролю вместо Basic Auth, а также есть функция сброса пароля.

## Как использовать

### Первый запуск

1. Запустите сервер: `node index.js`
2. Откройте в браузере: `http://localhost:3000/setup`
3. Создайте первого администратора, указав email и пароль (минимум 6 символов)
4. После создания вы автоматически будете авторизованы

### Вход в систему

1. Откройте: `http://localhost:3000/login`
2. Введите email и пароль
3. После входа вы будете перенаправлены на главную страницу

### Забыли пароль?

1. На странице входа нажмите "Забыли пароль?"
2. Введите ваш email
3. Проверьте почту - на неё придет ссылка для сброса пароля
4. Перейдите по ссылке и введите новый пароль

### Выход из системы

На главной странице админки есть кнопка "Выйти" в правом верхнем углу.

## Технические детали

- Пароли хранятся в зашифрованном виде (bcrypt)
- Сессии действуют 24 часа
- Данные пользователей хранятся в `data/users.json`
- Токены сброса пароля действуют 1 час
- Basic Auth все еще работает для обратной совместимости

## Переменные окружения

### Основные
- `SESSION_SECRET` - секретный ключ для сессий (рекомендуется изменить)
- `ADMIN_USER` - логин для Basic Auth (для обратной совместимости)
- `ADMIN_PASS` - пароль для Basic Auth (для обратной совместимости)

### Email (для сброса пароля)
- `EMAIL_HOST` - SMTP сервер (например: smtp.gmail.com)
- `EMAIL_PORT` - порт SMTP (обычно 587)
- `EMAIL_USER` - email для отправки
- `EMAIL_PASS` - пароль email
- `EMAIL_FROM` - от кого отправлять (по умолчанию = EMAIL_USER)
- `BASE_URL` - базовый URL сайта (для ссылок в письмах)

## Настройка Email

См. файл `EMAIL_SETUP.md` для подробных инструкций по настройке email.

## Безопасность

- Для production рекомендуется использовать HTTPS
- Измените `SESSION_SECRET` на случайную строку
- Используйте сильные пароли
- Настройте email для безопасного сброса паролей
